# xc-devflow-skills

## 项目概述

xc-devflow-skills 是一个基于 AI 的产品开发全流程协作管理系统，通过定义产品经理（PM）、研发（RD）和测试（QA）三个核心角色的智能体（Agent）工作规范，实现需求从构思到交付的完整自动化流程。

## 核心特性

### 🎯 三角色协作体系
- **产品经理（PM）**：负责需求分析、文档编写、产品规划
- **研发（RD）**：负责技术方案设计、代码实现、功能开发
- **测试（QA）**：负责测试方案制定、测试执行、Bug 管理

### 📋 标准化工作流程
- **PM 工作流**：研究分析 → 构思设计 → 编写文档 → 优化完善
- **RD 工作流**：研究需求 → 构思方案 → 编写计划 → 审核计划 → 执行计划 → 归档留痕
- **QA 工作流**：读取需求 → 读取开发成果 → 制定测试方案 → 编写测试计划 → 执行测试 → 提交Bug

### 🔄 审核机制
关键阶段设置人工审核节点，确保质量可控：
- PM：研究分析阶段需审核
- QA：制定测试方案阶段需审核
- RD：根因分析、修复方案需审核

### 📁 规范化文档管理
- 统一的文档结构规范
- CSV 格式的任务流转追踪
- Markdown 格式的需求文档

## 使用说明

### 入口交互

根据不同需求选择对应角色入口：

| 命令 | 角色 | 功能 |
|------|------|------|
| `/pm` | 产品经理 | 新建或修改需求文档 |
| `/rd` | 研发 | 开发需求功能 |
| `/fix` | 研发 | 修复 Bug |
| `/qa` | 测试 | 执行测试流程 |

### 工作流程详解

#### 产品经理（PM）
1. **研究分析** ⏸️ 需审核
2. **构思设计** ▶️ 自动执行
3. **编写文档** ▶️ 自动执行
4. **优化完善** ▶️ 自动执行

#### 研发人员（RD）
1. 研究需求文档
2. 构思技术方案
3. 编写开发计划
4. 审核计划（需人工确认）
5. 执行开发计划
6. 归档留痕

#### 测试人员（QA）
1. 读取需求文档
2. 读取开发成果
3. 制定测试方案 ⏸️ 需审核
4. 编写测试计划/脚本
5. 执行测试
6. 提交 Bug + 输出报告

## 角色规范

### 产品经理规范
- 输出结构化的需求文档（PRD）
- 确保需求清晰、可测试、可实现
- 遵循文档规范，输出符合标准格式

### 研发规范
- 代码必须与需求紧密结合
- 禁止引入技术负债
- 遵循文件修改规则
- 服务启停需遵守规范

### 测试规范
- 严格遵循测试范围铁律
- 禁止引入技术负债
- 规范使用 CSV 进行 Bug 追踪
- 完整记录测试上下文

## CSV 文件规范

### 需求索引（pm/prd/index.csv）
记录所有需求文档的索引信息。

### 待处理队列（rd/bugs/pending.csv）
QA 写入待修复的 Bug，RD 读取处理。

### 已归档记录（rd/bugs/archived.csv）
RD 写入已修复验收的 Bug 记录。

### 待测试队列（qa/pending.csv）
QA 写入待测试的任务清单。
