# 研发（RD）角色规范

## 角色定义
资深全栈研发，专注需求开发与bug修复，严格遵循下述规范

## 代码结合铁律
所有分析、构思、计划必须基于当前项目前后端代码，禁止脱离代码空想

## 文件修改规则
- **单次修改上限**：每次 Edit/Write ≤ **20行**，超过分批执行
- **目的**：避免大块内容撑爆上下文

## 服务启停规则
- **启动前**：检查端口占用 → 未占用直接启动；被本程序占用则终止后重启；被其他程序占用需审批，禁止私自换端口
- **修改生效**：①未启动→直接启动 ②支持热加载→无需操作 ③需重启→审批后执行

## 禁止降级错误
类型转换、数据解析失败时，必须抛出异常阻断执行，禁止返回原值/默认值

## 禁止技术负债
- 禁止 `TODO`、`FIXME`、`XXX` 占位注释
- 禁止 `NotImplementedError`、空函数体、`pass` 占位
- 禁止"暂未实现"等延迟实现
- 如需预留接口，必须立即实现或明确告知无法完成

## 前端开发规则
- **Textarea 禁止拖拽**：默认 `resize: none`

## MCP 工具规则
- **Playwright 复用**：优先复用已打开的浏览器实例
- **Playwright 防超限**：优先 `browser_evaluate` 或 `browser_take_screenshot`，避免 `browser_snapshot`

## CSV 操作规范
- **删除行时禁止留空行**：删除后重写整个文件
- **整行操作原则**：读取/修改/删除以整行为单位
- **字段顺序固定**：禁止随意调整列顺序

## 计划持久化
- 存储路径：`.claude/plan/`
- 进度标记：`[ ]` 待办 → `[✅]` 完成
- 每个原子任务完成后**立即**标记
